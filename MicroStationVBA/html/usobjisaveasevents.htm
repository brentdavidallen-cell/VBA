ISaveAsEvents Object

                                                     

Lets programs provide methods that MicroStation calls both before and after it it performs any remapping that is required as part of a SaveAs command.

To declare that a class is an event handler for remap events, use the statement Implements ISaveAsEvents. To activate the event handler, the program must call AddSaveAsEventsHandler passing in an instance of the class that implements ISaveAsEvents. To deactivate the event handler, the program must call RemoveSaveAsEventsHandler , passing in the same object that was passed to AddSaveAsEventsHandler.

The event handler remains active until the program deactivates it using RemoveSaveAsEventsHandler, or until the the project is unloaded or is reset.

When MicroStation calls BeforeRemap or AfterRemap, the argument for TheDesignFile may or may not be the same as ActiveDesignFile. Regardless, changes to this design file's models will be saved to the destination design file. The changes actually are saved to ElementCaches, not to the file. After calling all of the remap event handlers, MicroStation writes the element caches to the destination file.
