ModelReference Object

                                                     

All graphical elements in MicroStation are in models. A Visual Basic program has to access a model to be able to save or retrieve graphical elements.

Remarks

A MicroStation session may access many models. The models may be in different design files. The same model may be referenced different ways. For example, a model may be in use as the active model. There may also be different views of the model mapped into the same view. The MicroStation object model treats each of these references as a ModelReference object.

global ActiveModelReference gives you a ModelReference to whichever model is currently considered to be the "active" one. Typically, most of your model interactions will be with ActiveModelReference.

Some ModelReference objects are also Attachment objects. That is, they represent models that are referenced from within other models.

An Attachment object includes all of the methods and properties of ModelReference, plus additional methods and properties that pertain to the nature of the attachment, itself.

To determine whether a ModelReference object is also an Attachment object, use the IsAttachment property. To obtain a ModelReference object as an Attachment object, use the AsAttachment property.
