CommandState Object

                                                     

The methods of the CommandState object are used to implement commands that behave like standard MicroStation element-creation and element-modification commands. Such commands should implement a class that implements either IPrimitiveCommandEvents or ILocateCommandEvents. Use IPrimitiveCommandEvents if the new command is an element-creation command. Use ILocateCommandEvents if the new command is an element-modification command. Such commands typically run until the user invokes another command. However, if the command does need to terminate itself, it should call StartDefaultCommand to set MicroStation back to a common state.

MicroStation's commands can be classified as view manipulation commands, settings commands, element creation and modification commands, and utility commands.

The settings commands happen without additional user input. They typically are used to modify an active setting such as ACTIVE COLOR or ACTIVE LEVEL. These commands can occur while an element creation/modification command is active. The element creation/modification command remains active and in the same state after the settings command completes.

The view manipulation commands control what is displayed in a view. Examples are FIT and WINDOW AREA . Often, a view manipulation command requires user events such as data points. If so, the view manipulation command suspends any active element creation/modification command. Once the command is complete, the element creation/modification command resumes in the same state.

The element creation/modification commands are the commands that actually make changes to the model. These are categorized as primitive commands and locate commands. The primitive commands are the commands that create new elements. The locate commands are the commands that modify elements. A class that implements a new element-creation command should implement the IPrimitiveCommandEvents interface. A class that implements a new element-modification command should implement ILocateCommandEvents. Element-creation commands and element-modification commands remain active until a utility commands runs or until another element-creation/modification command runs.

Utility commands have a significant effect on the MicroStation session. COMPRESS DESIGN is an example of a utility command. Such commands must set MicroStation to a known state prior to executing. To do so, these commands should call the method StartDefaultCommand.
