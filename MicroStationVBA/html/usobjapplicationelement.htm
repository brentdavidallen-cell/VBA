ApplicationElement Object

                                                     

It is sometimes useful to add custom-formatted, non-graphical data to a design file. An application can use an ApplicationElement to save non-graphical data that is not associated with a specific element, or user attribute data (see AddUserAttributeData) for non-graphical data that is specific to an element. Both ApplicationElements and user attribute data provide means for an application to add its own custom data. The structure and meaning of the data is controlled entirely by the application, itself.

A design file can contain any number of ApplicationElements, from any number of applications. Therefore, each application needs a way to distinguish "its own" custom data from those of other applications (including those of MicroStation, itself). That's the purpose of the ApplicationID field. If you fail to use a unique ApplicationID, then your application may erroneously access, delete, or overwrite MicroStation's data or some other application's data, or MicroStation or some other application may erroneously access, delete, or overwrite your data.

To avoid potential conflicts worldwide, application IDs are managed by Bentley Systems, Inc. Please contact developer support at Bentley Systems to reserve an ID that is unique to your organization. (There is no charge for this service, and no questions will be asked regarding the purpose or use of your custom data.)

If your organization uses more than one kind of custom data, you can introduce a "subtype" field within your data structure, to distinguish the different types of custom data within your organization.

A VBA macro can add an ApplicationElement to a DesignFile by creating a DataBlock object, calling CreateApplicationElement to create the element and transfer the data from the DataBlock to the ApplicationElement, and then calling AddElement to save the element to the design file. CreateApplicationElement copies the entire DataBlock to the Element. MicroStation does not interpret the contents of the DataBlock.
