Type Property

                          

Read-only MsdElementType for ApplicationElement, ArcElement, AuxiliaryCoordinateSystemElement, BsplineCurveElement, BsplineSurfaceElement, CellElement, ComplexShapeElement, ComplexStringElement, ConeElement, CurveElement, DimensionElement, Element, EllipseElement, LineElement, MultiLineElement, NamedGroupElement, PointStringElement, SavedViewElement, ShapeElement, SharedCellDefinitionElement, SharedCellElement, SmartSolidElement, TagElement, TextElement, TextNodeElement.

Read/Write MsdModelType for Attachment, ModelReference.

Read-only MsdFontType for Font.

Read-only MsdRasterBlockType for RasterBlockType.

Gets (and, for some objects, sets) a value that identifies this object's fundamental type.

Syntax

object.Type

The Type property syntax has these parts:

Part Description
object A valid object.
MsdModelType An MsdModelType expression.

Remarks

The significance of "fundamental type" depends on the object:

For an element, the value (of the type MsdElementType) indicates which type of element it is: ArcElement, TextElement, etc.

Element.Type is an alternative to the various Element.Is... properties (such as Element.IsArcElement).

Generally, the Element.Is... property is preferred when testing one or two element types with If...Then statements. Element.Type is preferred when testing a large number of element types with a Select Case statement.

For a ModelReference object, the value (of the type MsdModelType) indicates whether it is a normal model, a sheet model, or an extraction model.

For a Font object, the value (of the type MsdFontType) indicates whether it is a MicroStation native font, an AutoCAD SHX font, or a Windows TrueType font.

For a RasterBlockType, gets the MsdRasterBlockType value associated with the block type.
