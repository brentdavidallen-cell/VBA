Weight Property

                          

Read/Write Double.

Gets or sets the weight of this object at the given valid index or indices.

Syntax

object.Weight (Index)[= Double]

The Weight property syntax has these parts:

Part Description
object A valid object.
Index A Long expression. For BsplineCurve objects, the zero-based index (between 0 and PolesCount-1, inclusive).
VIndex A Long expression. For BsplineSurface objects, the zero-based row index (between 0 and VPolesCount-1, inclusive).
UIndex A Long expression. For BsplineSurface objects, the zero-based column index (between 0 and UPolesCount-1, inclusive).
Double A Double expression.

Remarks

For BsplineCurve and BsplineSurface objects, each weight is a shape parameter associated with a Pole: the larger the weight relative to the other weights, the more the B-spline curve or surface is pulled toward the corresponding pole.

For BsplineCurve and BsplineSurface objects, weights are positive values less than or equal to 1.0. All other things being equal, an object with no weights is equivalent to one whose weights are all the same.

For BsplineCurve and BsplineSurface objects, the object does not have to be Rational before setting this property; if a valid weight less than 1.0 is set, then the object becomes Rational, and the remaining weights are populated with the value 1.0. On the other hand, setting a weight equal to all the other weights makes the object nonRational.

For BsplineCurve objects, weights are stored in a 1-dimensional array with PolesCount entries. This array is parallel to the Pole array.

For BsplineSurface objects, weights are stored in a 2-dimensional array with VPolesCount rows and UPolesCount columns. This array is parallel to the Pole array, thus it has row-major ordering: the second parameter varies faster (along a row). For example, Weight(v,u) refers to the weight in (zero-based) row v and column u.
