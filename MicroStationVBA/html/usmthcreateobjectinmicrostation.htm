CreateObjectInMicroStation Method

                          

Creates an object in MicroStation's address space.

Syntax

Set Object = object.CreateObjectInMicroStation (ProgID)

The CreateObjectInMicroStation method syntax has these parts:

Part Description
object A valid object.
ProgID A String expression.

Remarks

CreateObjectInMicroStation calls the Win32 function CLSIDFromProgID to translate ProgID into a CLSID, and then uses that CLSID as an argument to the Win32 function CoCreateInstance to create the object.

CreateObjectInMicroStation is useful for an out-of-proc program that needs to create objects from a server that must run in MicroStation's address space.

A program that needs to create a CopyContext, DataBlock, or ElementScanCriteria object in a separate process should use an expression like app.CreateObjectInMicroStation ("MicroStationDGN.CopyContext") where app is an object reference to the Application object for that process. Creating the object this way guarantees that the object is created in the correct process.

Using an expression like new CopyElement in a VBA program running in MicroStation works correctly to create a CopyContext object in the current process. Therefore, it is the proper thing to do in a VBA application. However, using the expression new CopyElement to create a CopyContext object from a separate process while there are multiple copies of MicroStation running may create an object for the wrong process.
