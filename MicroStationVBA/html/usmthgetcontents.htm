GetContents Method

                          

Creates and returns an ElementEnumerator that represents the set of elements defined by the the fence.

This method can change the contents of the design file. See Remarks for more information.

Syntax

Set ElementEnumerator = object.GetContents [(CloneElementsIfClipping [, IncludeLockedElements])]

The GetContents method syntax has these parts:

Part Description
object A valid object.
IncludeLockedElements Optional. A Boolean expression. Controls whether read-only elements (elements that are locked and elements that are from attachments) are included in the result. It is common to specify True if the elements will not be modified. The default value is True.
CloneElementsIfClipping Optional. A Boolean expression. How to handle the elements if fence clipping is turned on. The default value is False. See Remarks for more information.

Remarks

The behavior of GetContents depends on several factors, as summarized in the following table. The table uses the following terminology:

  • The fence containment area is the graphical region defined by the fence.

    If ActiveSettings.FenceVoid = False, the fence containment area is the interior of fence boundary.

    If ActiveSettings.FenceVoid = True, the fence containment area is the exterior of the fence boundary (that is, everything except the interior).

  • An element is wholly contained if it lies entirely within the fence containment area.
  • An element is partially contained if it overlaps the fence boundary.
Value of ActiveSettings .FenceClip Value of ActiveSettings .FenceOverlap Value of GetContents' argument, CloneElementsIfClipping Effect of calling GetContents
False False (irrelevant) GetContents does not change or create any elements. It returns all of (and only) those elements that are wholly contained.
False True (irrelevant) GetContents does not change or create any elements. It returns all of (and only) those elements that are either wholly or partially contained.
True (irrelevant) False GetContents does not change any wholly-contained elements. However, for each partially-contained element, it slices that element at the fence boundary into two or more smaller elements, which replace the original element in the design file. It then returns all contained elements, including those that are a result of slicing.
True (irrelevant) True GetContents does not change any existing elements, but it creates new elements as follows: It copies (clones) each wholly-contained element. For each partially-contained element, it creates one or more smaller elements that represent only the contained portion(s) of the original element. It adds all of the newly-created elements to the design file, and returns them.
