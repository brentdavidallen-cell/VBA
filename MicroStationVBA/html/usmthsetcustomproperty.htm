SetCustomProperty Method

                          

Sets a custom property in the print definition that will be created or updated from the collected design state.

Syntax

For the DesignFile object:

   object.SetCustomProperty Name, Value

For the PrintDefCollectStateParam object:

   object.SetCustomProperty name, value

The SetCustomProperty method syntax has these parts:

Part Description
object A valid object.
name A String expression. Custom property name.
value A String expression. Custom property value. If empty, custom property is undefined.
Name A String expression.
Value A VARIANT expression.

Remarks

SetCustomProperty decides on the data type to save based on the datatype of the value in the variant parameter Value. It can save any of the integer types, String, Boolean, Single, Double, or Date. If the file does not already have custom, SetCustomProperty creates it. If custom already exists but the type is different than the type of data being stored, SetCustomProperty changes the property's data type to match Value's data type.

You can also examine or modify custom properties using a tool like Windows Explorer.
