AddUsingNamedView1 Method

                          

Attahes a model reference, using a named view for clipping information and for reference origin.

Syntax

Set Attachment = object.AddUsingNamedView1 (FileSpecification, ModelName, LogicalName, Description, ViewName, CenterPoint, Flags)

The AddUsingNamedView1 method syntax has these parts:

Part Description
object A valid object.
FileSpecification A String expression. The name of the file that contains the model being attached. The drive, directory name, and extension are all optional.
ModelName A String expression. ModelName is ignored unless ViewName refers to a standard view such as "Top" or "Left". Every saved view includes a model name, and AddUsingNamedView1 cannot override that. However, a standard view name does not imply a specific model.
LogicalName A String expression. The logical name for the attachment. The logical name must be unique.
Description A String expression. The description that is displayed in the references dialog box.
ViewName A String expression. The name of the view. This may be the name of a saved view or a standard view. The standard view names are Top, Bottom, Left, Right, Front, Back, Isometric, and Right Isometric. See SavedViewElement for more information on saved views.
CenterPoint A Point3d expression. Specifies a position in the master file where the named view will be centered. Specified in master units.
Flags An MsdAddAttachmentFlags expression. Or together flags to specify multiple values.

Remarks

The format of FileSpecification is configuration_variable:filename where the configuration_variable: part is optional. The configuration_variable part and the colon must appear together. Do not use one without using the other.

If FileSpecification specifies configuration_variable: then MicroStation records configuration_variable: as part of the Attachment. It then evaluates the configuration variable everytime MicroStation loads the Attachment.

If FileSpecification does not provide a full file specification, the master file path and the environment variables MS_RFDIR and MS_DEF are searched sequentially for the file.

Specify msdAttachmentFlagTrueScale for Flags to cause the attachment to be true scale.

Specify msdAttachElementsVisible to cause MicroStation to immediately display the attachment. If the program does not specify msdAttachElementsVisible, then it should subsequently set ElementsVisible to True and then call both Rewrite and Redraw.

Attachments.AddUsingNamedView1 does not need to have the model specified because the named view is always a view of a specific model.
