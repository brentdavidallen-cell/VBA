NamedGroupMember Object

                                                     

A NamedGroupMember object contains the information that controls how an Element behaves in a specific named group. This object type is necessary because an Element can be in more that one named group, and the behavior can be different in each of the groups.

Changes to a NamedGroupMember object are saved to the model only when the corresponding NamedGroupElement is saved to the model. Therefore, a program should rewrite the NamedGroupElement after making changes to the NamedGroupMember. For example, if the variable member holds a reference to a NamedGroupMember object and the program uses that reference to change the member, then the program could commit the changes to the file with the statement member.ParentGroup.Rewrite.
