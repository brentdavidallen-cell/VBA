Element Symbology, ByLevel Symbology, and Override Symbology

When drawing an Element in a view, MicroStation must determine what color, line style, and line weight to use. To do so, it first examines the view's DisplaysLevelSymbology property. If it is True, MicroStation examines the override symbology from Element.Level. It uses OverrideColor unless UsingOverrideColor is False, OverrideLineStyle unless UsingOverrideLineStyle is False, and OverrideLineWeight unless UsingOverrideLineWeight is False.

If the view's DisplaysLevelSymbology property is False or the UsingOverride... property is False, then MicroStation must decide whether to take the symbology directly from the element, or to use the symbology specified with the level. If Element.Color is ByLevelColor, then MicroStation draws the element using the Element.Level.ElementColor. If Element.Color is ByCellColor, then MicroStation draws the element using the cell's color. Otherwise, MicroStation uses the element's color. Similar logic applies for the line weight and line style.

A program can use ApparentColor, ApparentLineWeight, and ApparentLineStyle to determine what color, line style, and line weight MicroStation will use to draw an element.
