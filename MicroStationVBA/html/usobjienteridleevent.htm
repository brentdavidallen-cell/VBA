IEnterIdleEvent Object

                                                     

Defines the methods MicroStation uses to notify a program of IEnterIdleEvent events.

Remarks

To receive these events, a program must have a class that implements IEnterIdleEvent. The program must create an instance of the class, and register it with MicroStation by calling AddEnterIdleEventHandler.

Since a new MicroStation process does not enter the idle state until it has finished initializing, a program can use this event to be notified when initialization is complete. This is sometimes very important for a program that starts a new MicroStation process.

This event can also be useful for an OnDesignFileOpened event handler that needs to be notified the next time MicroStation becomes idle.

MicroStation performs a lot of work based on timer events even when there is no user input. After every timer event, MicroStation briefly returns to the idle state causing it to call the enter-idle event handlers again. Consequently, it will call the enter-idle event handlers very frequently. To prevent this from happening, a normal enter-idle event handler should remove itself the first time it is called.

MicroStation does not call the EnterIdle event handlers when it is waiting for input for a modal dialog box. For example, it does not call EnterIdle event handlers while the file open dialog box is open.
