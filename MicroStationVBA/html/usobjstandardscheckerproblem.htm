StandardsCheckerProblem Object

                                                     

A StandardsCheckerProblem object represents a Problem node in the StandardsCheckerController's report.

Remarks

A standards checker can use StandardsCheckerReport.AddProblem or AddIgnoredProblem to add a problem to a report. Both of these methods return a StandardsCheckerProblem object that the standards checker can use to add more information to the entry in the report.

A problem entry in the report looks like: 

    <Problem fileId="1" modelId="0" appId="bentley.com.ByLevel" ignored="1"
                                ignoredBy="Larry LeRue" ignoredAt="2003/10/29 10:51:20" fixed="0" type="Element">
      <Description>The element does not meet the 'ByLevel' requirements</Description>
      <Standard>Line</Standard>
      <ElemId>788</ElemId>
      <Variances>
         <Variance Property="Color" expected="ByLevel" found=" 12"/>
         <Variance Property="Style" expected="ByLevel" found="0"/>
      </Variances>
    </Problem>

The AddProblem methods insert the fileID and modelID that corresponds to the ModelReference argument passed into RunCheck. The AddProblem methods use the standards checker's IdentityString property to get the value for appID. The standards checker provides the values for Ignored, IgnoredBy, and IgnoredWhen as arguments to AddIgnoredProblem. The standards checker provides a value for Description as an argument to the AddProblem method. The standards checker provides a value for type via the ObjectType parameter of the AddProblem method.

A standards checker can call AddAction, AddElementID, AddSeverity, AddStandard, or AddVariance to add detail to the report. It can call these methods in any order. These methods keep the nodes in the proper order.
