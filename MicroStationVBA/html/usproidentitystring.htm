IdentityString Property

                          

Read-only String.

The IdentityString provides a short name that uniquely identifies the Standards Checker handler.

Syntax

object.IdentityString

The IdentityString property syntax has these parts:

Part Description
object A valid object.

Remarks

This name appears in the StandardsCheckerController's report in places where it is necessary to identify what application created a segment of the report. The user of the report will never see this name unless he examines the raw XML report itself.

MicroStation uses this as a node in an XML document, and uses it in XPath expressions to search the document Since MicroStation uses IdentityString as an XML node name, it must satifisfy the requirements of a valid XML node name. Therefore, it must begin with a letter. Every subsequent character in the identifier must be a letter, number, period (.), hyphen (-), or underscore.

If IdentifyString is not a valid XML node name, MicroStation generates a valid node name by replacing each invalid character with an underscore. If the program passes in "True Type Test", MicroStation uses "True_Type_Test".
