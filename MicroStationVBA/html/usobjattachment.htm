Attachment Object

                                                     

Represents a model that is referenced within (attached to) another model.

Remarks

Since an Attachment is a kind of a ModelReference, it includes all of the methods and properties of ModelReference. However, an Attachment is a read-only ModelReference that cannot become the active model reference, so many of the methods raise errors. For example, Activate raises an error because an Attachment cannot be activated, and AddElement raises an error because an Attachment is read-only.

It also includes additional methods and properties that pertain to the nature of the attachment itself.

Changing the ElementsVisible property is equivalent to changing the Display column in the References dialog box. Changing ElementsLocatable is equivalent to changing Locate, and changing ElementsSnappable is equivalent to changing Snap.

Call Rewrite after changing an Attachments attributes to make the change permanent.
