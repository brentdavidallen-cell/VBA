Dynamics Event

                          

MicroStation calls the Dynamics event handler when dynamics is active and the user moves the cursor. The command should use this event handler to display the element as it would appear if the user entered a data point at the current cursor location.

Syntax

object.Dynamics Point, View, DrawMode

The Dynamics method syntax has these parts:

Part Description
object A valid object.
Point A Point3d expression. The position of the cursor.
View A View expression.
DrawMode An MsdDrawingMode expression.

Remarks

When MicroStation starts a primitive command, it turns off dynamics. Dynamics remains off until the program calls CommandState.StartDynamics. A command ordinarily does not start dynamics until at least one data point has been entered.

When displaying the element, the command must specify the value of the DrawMode parameter as the input into Redraw. For each cursor movement, MicroStation calls the Dynamics event handler twice. It calls it once to erase what the handler drew the last time it was called, and then it calls it to draw the element using the new point. If the application fails to erase exactly the same element as it drew, then garbage appears on the screen. This is not a problem if the event handler uses the value supplied in Point to create an element. It is only a problem if the event handler modifies Point in some way, or creates the element with different symbology.
