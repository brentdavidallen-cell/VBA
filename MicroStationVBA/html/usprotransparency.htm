Transparency Property

                          

Read/Write Double for ApplicationElement, ArcElement, Attachment, AuxiliaryCoordinateSystemElement, BsplineCurveElement, BsplineSurfaceElement, CellElement, ChainableElement, ComplexShapeElement, ComplexStringElement, ConeElement, CurveElement, DimensionElement, Element, EllipseElement, IntersectableElement, LineElement, MultiLineElement, NamedGroupElement, PointStringElement, SavedViewElement, Settings, ShapeElement, SharedCellDefinitionElement, SharedCellElement, SmartSolidElement, TagElement, TextElement, TextNodeElement, TraversableElement.

Read/Write Long for PrintDisplayParams.

Read/Write Boolean for RenderingInformation.

Gets or sets the transparency state.

Syntax

object.Transparency [= Double]

The Transparency property syntax has these parts:

Part Description
object A valid object.
Double A Double expression.
value A Long expression.
Boolean A Boolean expression.

Remarks

For RenderingInformation, Transparency is a Boolean that controls whether tranparency is displayed. For Attachment and for the element types, Transparency controls the transparency of the object. The transparency is 0 for a completely opaque object, and 1.0 for a completely transparent object. If the new value is less than 0, the object's transparency is set to 0. If it is greater than 1, the objec's transparency is set to 1.

For an element, getting or setting the value of Transparency raises an error if the element is not graphical. If you're in doubt, firstly check the element's IsGraphical property.

Transparency is ignored for cells and shared cells. The display process does interpret the transparency of the components, but the display process does not use the transparency of the header to adjust the transparency of the components.
