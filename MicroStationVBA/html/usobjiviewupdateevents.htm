IViewUpdateEvents Object

                                                     

Defines the methods MicroStation uses to notify a program of View update events.

Remarks

To receive these events, a program must have a class that implements IViewUpdateEventsHandler. The program must create an instance of the class, and register it with MicroStation by calling AddViewUpdateEventsHandler.

MicroStation calls BeforeRedraw after erasing the Views and prior to drawing the contents of the Views. If calls AfterRedraw after drawing the contents of the Views.

Both BeforeRedraw and AfterRedraw receive an argument TheViews that is an array of views that are being updated.

Both methods also receive an argument TheModels. For a normal update, TheModels is a zero-length array and the update affects all models. For an operation on a reference attachment, TheModels contains a list of Attachments that are being redrawn.

Both methods get a DrawMode argument that tells how the elements of the model are being redrawn. For a normal update, this is msdDrawingModeNormal. It is msdDrawingModeHilite when MicroStation is highlighting the elements prior to asking the user to confirm that he wants to detach the reference. It is msdDrawingModeErase when MicroStation is erasing the elements of a reference attachment. This can happen when the user detaches the reference, or when the user turns off display for the reference attachment.

Warning

Using the IViewUpdateEvents to draw elements that have not been added to a model does not work well in views that have Graphics Acceleration enabled.
