RunCheck Method

                          

Tells the Standards Checker to check the data in the specified model.

Syntax

object.RunCheck ModelToCheck, FirstModel, Options

The RunCheck method syntax has these parts:

Part Description
object A valid object.
Options A Long expression. A value constructed by Or'ing together values from MsdStandardsCheckerOptions.
ModelToCheck A ModelReference expression.
FirstModel A Boolean expression.

Remarks

See StandardsCheckerController for more information on the organization of a standards checker.

A standards checker can ignore the values passed in Options. The StandardsCheckerController remembers the value that it passes in Options. All of the methods of StandardsCheckerController have different behavior based on these values. For example, if Options does not have msdStandardsCheckerCheckOptionFixErrors Or'ed in then ShowCheckerError will not return msdStandardsCheckerReplaceChoiceFix.

A standards checker typically scans through a model. When it finds an error, it must first display the error to the user. Then it must act on the users response. Finally, it must add the error to the report. The standards checker can use ShowCheckerError or ShowCheckerErrorWithFixOptions to display the error. The standards checker can use AddProblem or AddIgnoredProblem to the report.

The standards checker must also increment the error counts when it encounters an error. The error counts are FixedProblems, IgnoredProblems, and TotalProblems.
