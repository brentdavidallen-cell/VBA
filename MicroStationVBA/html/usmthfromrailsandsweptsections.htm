FromRailsAndSweptSections Method

                          

Initializes this object by sweeping the given start/end sections along the given rail path(s).

Syntax

object.FromRailsAndSweptSections Rail0, Rail1, Section0, Section1 [, SectionsCount [, Tolerance [, SkinDirection [, RotateSections [, RotationAxisFromRail0 [, ForceOrthogonality [, ScaleSections [, ScaleUniformly]]]]]]]]

The FromRailsAndSweptSections method syntax has these parts:

Part Description
object A valid object.
Rail0 A BsplineCurve expression. A path that forms the v(u)=0 isoparametric curve of this object.
Rail1 A BsplineCurve expression. A path that forms the v(u)=1 isoparametric curve of this object. Pass Nothing for 1-rail sweep.
Section0 A BsplineCurve expression. The cross-sectional initial profile that forms the u(v)=0 isoparametric curve of this object. For best results, Section0 should be in position and planar.
Section1 A BsplineCurve expression. The cross-sectional final profile that forms the u(v)=1 isoparametric curve of this object. For best results, Section1 should be in position and planar. Pass Nothing for 1-section sweep; otherwise, Section1 is swept backwards over the rail(s) and these sections are averaged with those generated by sweeping Section0 forwards.
SectionsCount Optional. A Long expression. The number of sections to use in the construction, including Section0, Section1 and generated sections. If negative, then a default value of 10 times Rail0.PolesCount is used, but if Rail1 is given, then 5 times the sum of Rail0.PolesCount and Rail1.PolesCount is used in general.
Tolerance Optional. A Double expression. The absolute distance tolerance for section approximation, knot removal and point equality. If omitted or negative, a default tolerance of 1.0e-5 times the largest coordinate difference of the unweighted Poles of the section(s) is used. If zero, the swept sections are interpolated, no knots are removed, and the aforementioned default tolerance is used for point equality.
SkinDirection Optional. An MsdBsplineSurfaceDirection expression. The parameter direction for skinning. The generated sections are constant-u(v) curves if the direction is msdBsplineSurfaceU (msdBsplineSurfaceV). If omitted, a default direction of msdBsplineSurfaceV is used.
RotateSections Optional. A Boolean expression. Indicates whether the swept sections rotate to follow the rail(s). If False then sections do not rotate---if Rail1 is Nothing, this means that the frame of the initial section(s) is preserved. If omitted, a default value of True is used.
RotationAxisFromRail0 Optional. A Boolean expression. Indicates whether the swept section rotation axis is orthogonal to Rail0 tangents. If False then the swept section's rotation axis is its own x-axis. If omitted, a default value of True is used. This value is only relevant if Rail1 is Nothing and RotateSections is True.
ForceOrthogonality Optional. A Boolean expression. Indicates whether each given section is rotated orthogonal to Rail0 at its closer endpoint prior to sweeping. If False, there is no orthogonality enforcement. If omitted, a default value of False is used. This value is only relevant if Rail1 is Nothing.
ScaleSections Optional. A Boolean expression. Indicates whether the swept sections are scaled according to the distance between Rail0 and Rail1. If False, the sections are not scaled. If omitted, a default value of True is used. This value is only relevant if Rail1 is given.
ScaleUniformly Optional. A Boolean expression. Indicates whether the swept sections are scaled uniformly. If False, each section is scaled only along its local x-axis. If omitted, a default value of True is used. This value is only relevant if Rail1 is given and ScaleSections is True.

Remarks

For BsplineSurface objects, the given rail(s) and section(s) will be opened, made compatible and reversed into proper relative orientation if necessary. For best results, these curves should be nonrational, G1-continuous (tangent direction continuity) and should meet the given section(s) at the future corners of this object.

For BsplineSurface objects, each given section's origin is taken to be its closest point to the appropriate Rail0 endpoint. The section's x-axis is then defined by its closest point to the analogous Rail1 endpoint, or if Rail1 is Nothing, by its point at parameter 1.0, or if the first/last section points are equal, by the furthest intersection point (from the section origin) between the section and the plane containing: the appropriate Rail0 endpoint, the section origin and the section normal at the section origin.

For BsplineSurface objects, if only Rail0 and Section0 are given, alternate methods are available: FromRailAndSweptSection and FromRailAndSweptRigidSection. The former special-cases linear and circular segments of the rail while the latter allows linear control of scaling and rotation. These alternate methods do not in general approximate the rail curve as accurately as this method does, but they may perform better on rails with corners, and on segments that do not intersect the rail.
