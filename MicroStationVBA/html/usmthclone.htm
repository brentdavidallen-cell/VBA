Clone Method

                          

Returns a new object that is a copy of this object.

Syntax

For the TextNodeElement object, TextElement object, TagElement object, SmartSolidElement object, SharedCellElement object, SharedCellDefinitionElement object, ShapeElement object, SavedViewElement object, PointStringElement object, NamedGroupElement object, MultiLineElement object, LineElement object, EllipseElement object, Element object, DimensionElement object, CurveElement object, ConeElement object, ComplexStringElement object, ComplexShapeElement object, CellElement object, BsplineSurfaceElement object, BsplineCurveElement object, AuxiliaryCoordinateSystemElement object, ArcElement object and ApplicationElement object:

   Set Element = object.Clone

For the ElementEnumerator object:

   Set ElementEnumerator = object.Clone

The Clone method syntax has these parts:

Part Description
object A valid object.

Remarks

In many cases, ModelReference.CopyElement is preferred because it can adjust levels, graphic group numbers, etc. when copying elements from one design file to another.

THIS APPEARS TO NOT BE TRUE The new element, like all newly-created elements, is an "orphan": It is not a member of any model.
