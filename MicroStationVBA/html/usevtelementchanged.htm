ElementChanged Event

                          

Event handler called whenever an element is directly changed in MicroStation.

Syntax

object.ElementChanged AfterChange, BeforeChange, Action, CantBeUndone

The ElementChanged method syntax has these parts:

Part Description
object A valid object.
AfterChange An Element expression.
BeforeChange An Element expression.
Action An MsdChangeTrackAction expression.
CantBeUndone A Boolean expression.

Remarks

Turning on the "CantBeUndone" flag also means that ALL previous changes become non-undoable as well, since it is not possible to "skip" undo states. From the user's perspective, this is a pretty drastic action and should be avoided if at all possible.

ElementChanged events handlers are called AFTER the change has been completed and the ElementCache already holds the new copy of the element.

Use AddChangeTrackEventsHandler and RemoveChangeTrackEventsHandler to add and remove a Change Track events handler. A Change Track events handler is an instance of a class that implements the IChangeTrackEvents interface.
