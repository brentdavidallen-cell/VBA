Reattach Method

                          

Immediately attaches to a different model or file. See AttachName, Reattach, and SetAttachNameDeferred for more information.

Syntax

Set Attachment = object.Reattach (FileName, ModelName)

The Reattach method syntax has these parts:

Part Description
object A valid object.
FileName A String expression. If vbNullString, then Reattach uses the file name of the currently attached model.
ModelName A String expression. if vbNullString, then Reattach uses the model name of the currently attached model. It is okay to pass vbNullString if the model is to remain the same.

Remarks

If the file is not found, then Reattach raises an error. If the model is not found, then Reattach makes the attachment refer to the default model.

The format of the FileName is configuration_variable:filename where the configuration_variable: part is optional. The configuration_variable part and the colon must appear together. Do not use one without using the other.

If the parameter specifies configuration_variable: then MicroStation records configuration_variable: as part of the Attachment. It then evaluates the configuration variable everytime MicroStation loads the Attachment.

If FileName does not provide a full file specification, the master file path and the environment variables MS_RFDIR and MS_DEF are searched sequentially for the file.

Reattach makes the current Attachment object obsolete, and replaces it with a new Attachment object. If the program needs to make additional changes to the attachment, it must use the Attachment object Reattach returns.
