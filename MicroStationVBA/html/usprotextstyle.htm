TextStyle Property

                          

Read/Write TextStyle.

Gets or sets the active text style.

Syntax

[Set] object.TextStyle [= TextStyle]

The TextStyle property syntax has these parts:

Part Description
object A valid object.
TextStyle A TextStyle expression.

Remarks

MicroStation creates the active text style by copying the text style properties from the named text style into a separate text style. Any changes the user makes to the active text style are made to this local copy, not to the master definition of the text style. To remain consistent with this behavior, ActiveSettings.TextStyle gets a text style that represents this local buffer. It does not get a text style that corresponds to an entry in the text styles collection. Changes to this text style do not alter the original text style. Changes to this text style only affect the active text style. Getting ActiveSettings.TextStyle and changing it is similar to starting Place Text, selecting a text style in the Tool Settings dialog, and then changing the font in the Place Text dialog box. Changing the font in the Place Text dialog box changes the font that the active command uses, but it does not change the font in the text style ActiveDesignFile.Fonts (ActiveSettings.TextStyle.Name). To change the font of the text style itself instead of changing the override in the active text style, use ActiveDesignFile.Fonts (ActiveSettings.TextStyle.Name) to get the master definition and change that.

The fact that ActiveSettings.TextStyle returns a reference to a working copy of the text style can lead to some surprising behavior. The following code fragment illustrates this: 


    Dim ts As TextStyle

    Set ts = ActiveSettings.TextStyle
    CadInputQueue.SendCommand "textstyle active Architectural"

    '  Displays Active text style is Architectural
    Debug.Print "Active text style is " & ts.Name

    CadInputQueue.SendCommand "textstyle active Engineering"

    '   Displays Active text style is Engineering
    Debug.Print "Active text style is " & ts.Name
