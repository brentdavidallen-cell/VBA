Pole Property

                          

Read/Write Point3d.

Gets or sets the pole of this object at the given valid index or indices.

Syntax

object.Pole (Index)[= Point3d]

The Pole property syntax has these parts:

Part Description
object A valid object.
Index A Long expression. For BsplineCurve objects, the zero-based index (between 0 and PolesCount-1, inclusive).
VIndex A Long expression. For BsplineSurface objects, the zero-based row index (between 0 and VPolesCount-1, inclusive).
UIndex A Long expression. For BsplineSurface objects, the zero-based column index (between 0 and UPolesCount-1, inclusive).
Point3d A Point3d expression.

Remarks

For BsplineCurve and BsplineSurface objects, the poles are control handles which have local influence over (but do not necessarily interpolate) the B-spline curve or surface. The B-spline curve or surface always lies in the convex hull of its poles.

For BsplineCurve and BsplineSurface objects, this property always gets/sets unweighted poles, even when the Rational property is True.

For BsplineCurve objects, poles are stored in a 1-dimensional array with PolesCount entries.

For BsplineCurve objects, if the Closed property is True, then the first and last poles do not have to be equal; if they are different, then the B-spline curve is periodically defined.

For BsplineCurve objects, to set a pole at an index beyond the valid index range, you must use SetPoles, which resets PolesCount.

For BsplineSurface objects, poles are stored in a 2-dimensional array with VPolesCount rows and UPolesCount columns. This array has row-major ordering: the second parameter varies faster (along a row). For example, Pole(v,u) refers to the pole in (zero-based) row v and column u.

For BsplineSurface objects, if the UClosed (VClosed) property is True, then the first and last poles in a row (column) do not have to be equal; if they are different, then the B-spline surface is periodically defined in the u-parameter (v-parameter) direction.

For BsplineSurface objects, to set a pole at indices beyond the valid index ranges, you must use SetPoles, which resets UPolesCount and VPolesCount.
