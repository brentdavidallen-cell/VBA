UpdateOrder Property

                          

Read/Write Long.

Gets or sets the update order.

Syntax

object.UpdateOrder [= Long]

The UpdateOrder property syntax has these parts:

Part Description
object A valid object.
Long A Long expression.

Remarks

Given 2 Attachments att1 and att2, if att1.UpdateOrder > att2.UpdateOrder then att1 is displayed after att2 is displayed and the elements in att1 cover the elements in att2.

The value for UpdateOrder is unique among a collection of Attachments. Therefore, when a program assigns a new value to an Attachment's UpdateOrder, the values for the other attachments are automatically adjusted.

By default, the first Attachment has UpdateOrder 1, Attachment n has UpdateOrder n. The model that the attachments are attached to has UpdateOrder 0. There is no way to explicitly access the update order of the root model, but a program can change it by changing the update order of the attachments.

If the program tries to set an UpdateOrder to a value less than 0, then the value is set to 0. If a program tries to set an UpdateOrder to a value greater than Attachments.Count, then the UpdateOrder is set to Attachments.Count.

Attachment.Rewrite does not save an Attachment's UpdateOrder. SaveSettings saves all of the UpdateOrder values in a group.
