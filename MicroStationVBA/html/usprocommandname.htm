CommandName Property

                          

Read/Write String.

Gets/sets the current command name.

Syntax

object.CommandName [= String]

The CommandName property syntax has these parts:

Part Description
object A valid object.
String A String expression.

Remarks

A program starting a command using StartPrimitive or StartLocate should set the CommandName property after calling StartPrimitive or StartLocate.

When a user starts a view-manipulation command, MicroStation saves the current value of CommandName prior to letting the view-manipulation command set its command name. MicroStation restores CommandName to the saved value when the view-manipulation command completes. That is not true for ShowCommand. ShowCommand just copies a string to the display area. That string is lost if the user starts and exits a view-manipulation command.

If a program uses CommandName to set the command name, then MicroStation associates that string with entries in the undo buffer when appropriate. Displaying a command name with ShowCommand does not cause MicroStation to associate that name with any entries in the undo buffer.
