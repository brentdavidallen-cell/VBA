Point3dInPolygonXY Method

                          

Returns a value representing whether a point is outside (-1), on (0), or inside (1) a polygon, ignoring z-coordinates.

Syntax

Long = object.Point3dInPolygonXY (Point, PolygonVertices [, Tolerance])

The Point3dInPolygonXY method syntax has these parts:

Part Description
object A valid object.
Point A Point3d expression. The point whose position is tested relative to the polygon.
PolygonVertices A Point3d () expression. The array of points comprising the polygon (first point does not have to equal last point).
Tolerance Optional. A Double expression. The absolute tolerance for determining if Point lies on the boundary of the polygon. If negative or omitted, a default tolerance of 1.0e-5 times the the maximum difference in the coordinates of PolygonVertices is used.

Remarks

Ignoring the z-coordinates of Point and PolygonVertices is equivalent to assuming that these points lie in a plane parallel to the xy-plane. If you are picking a polygon from a world point as seen through a View, you can meet this assumption by rotating both the Point and the PolygonVertices by the View.Rotation to rewrite these points in view coordinates. If your geometry lies in plane sPlane, then you can rotate Point and PolygonVertices into the xy-plane with the following code for each point sPoint: sPoint = Point3dFromTransform3dTimesPoint3d(Transform3dFromWorldToPlane3d(sPlane),sPoint).
