FindForDesignFile Method

                          

Searches given DesignFile for a ItemTypeLibrary whose name matches a given pattern. If found, returns the corresponding ItemTypeLibrary object; else returns Nothing.

Syntax

Set ItemTypeLibrary = object.FindForDesignFile (NamePattern, source [, includeDgnLibs [, PreviouslyFoundItemTypeLibrary]])

The FindForDesignFile method syntax has these parts:

Part Description
object A valid object.
NamePattern A String expression. The name of the ItemTypeLibrary(s) to find. This name follows the same rules as a pattern given to Visual Basic's Like operator. (Exception: The pattern match is always case-insensitive, regardless of the Option Compare setting.)
source An Object expression. Object of DesignFile, which contains required ItemTypeLibraies.
PreviouslyFoundItemTypeLibrary Optional. An ItemTypeLibrary expression. If Nothing (the default), Find returns the first ItemTypeLibrary whose name matches the given pattern. Otherwise, Find returns the next ItemTypeLibrary found after this one. Thus, you can call Find successively to find successive ItemTypeLibrarys whose names match the pattern.
includeDgnLibs Optional. A Boolean expression.

Remarks

You can find one specific ItemTypeLibrary by simply giving its name, without any wildcard characters.

You can successively find all ItemTypeLibraries by giving the name pattern, "*".
