ID Property

                          

Read-only DLong for ApplicationElement, ArcElement, AuxiliaryCoordinateSystemElement, BsplineCurveElement, BsplineSurfaceElement, CellElement, ComplexShapeElement, ComplexStringElement, ConeElement, CurveElement, DimensionElement, Element, EllipseElement, LineElement, MultiLineElement, NamedGroupElement, PointStringElement, SavedViewElement, ShapeElement, SharedCellDefinitionElement, SharedCellElement, SmartSolidElement, TagElement, TextElement, TextNodeElement.

Read-only Long for Font, Level, RasterColorMode, RasterCompressionMode, RasterFormat, TextStyle.

Read-only Int64 for Raster.

Gets a value that uniquely identifies this object.

Syntax

object.ID

The ID property syntax has these parts:

Part Description
object A valid object.

Remarks

For all element types, ID is a persistent value that is assigned when the element is first added to a model. It is unique across the entire DesignFile that contains the model. If an element has never been added to a model, then the element's ID is 0. When a program uses ModelReference.AddElement to add an element to the model and the element already has an ID, AddElement assigns a new ID. This happens when a program tries to add an element that is already in the file. Doing so creates 2 copies of the element with distinct ID's. If a progam deletes an element using ModelReference.RemoveElement and then adds that element using ModelReference.AddElement, then the element retains its ID. The ID of the deleted element is not reused, but if the program deletes the element and adds it the ID is retained.
