ShowCheckerErrorWithFixOptions Method

                          

Shows a problem description and lets the user specify how to handle the problem.

Syntax

object.ShowCheckerErrorWithFixOptions HandlingChoice, SelectedFix, ProblemDescription, FixesListBoxLabel, FixesColumnLabels, Fixes, DefaultFix, ReplaceOptions, CurrentlyIgnored [, IgnoredBy [, IgnoredWhen]]

The ShowCheckerErrorWithFixOptions method syntax has these parts:

Part Description
object A valid object.
HandlingChoice An MsdStandardsCheckerReplaceChoice expression. Output. Specifies how the user wants to handle the error.
SelectedFix A Long expression. Output. An index into the Fixes array. If Fixes is 1-based, then this value is 1-based. If Fixes is 0-based, then this value is 0-based.
ProblemDescription A String expression. Description displayed in the dialog box. "The Problem Description" in the following example.
FixesListBoxLabel A String expression. Label for the list of fixes. "Fixes List Box Label" in the following example.
FixesColumnLabels A String () expression. An array of labels. The array must be declared with 2 entries. It may be 0-based or 1-based, and either entry may be vbNullString or "".
Fixes A String () expression. An array of fixes. Each row corresponds to a row in the fixes section of the dialog box. This must be 2 columns wide. Any entry can contain a value to be displayed, can be left uninitialized, or can be set to vbNullString or "".
DefaultFix A Long expression. An index into Fixes.
ReplaceOptions An MsdStandardsCheckerReplaceOptions expression. Specifies what options are available and therefore determines what buttons are enabled in the dialog box. Options may be Or'ed together as in msdStandardsCheckerReplaceOptionCanFix Or msdStandardsCheckerReplaceOptionCanIgnore.
CurrentlyIgnored A Boolean expression. Set this to True if the error was previously ignored. If CurrentlyIgnored is True then ShowCheckerErrorWithFixOptions displays a Stop Ignoring button instead of the Ignore button and a Continue Ignoring button instead of the Skip button.
IgnoredBy Optional. A String expression. If CurrentlyIgnored is True then IgnoredBy and IgnoredWhen are displayed in the Tool Tips box when the cursor is over the Continue Ignoring or Stop Ignoring button.
IgnoredWhen Optional. A Date expression.

Remarks

The StandardsCheckerController opens a status dialog box that the current standards checker should use to display its status. The checker should call ShowCheckerStatus to display a status message in the status area of the dialog box. It should use ShowCheckerError or ShowCheckerErrorWithFixOptions to display a problem description along with the fix options in the middle of the dialog. ShowCheckerErrorWithFixOptions calls the checker's GetFixDetail method to get information to display in the bottom of the dialog box.

If the standards checker calls this function with CurrentlyIgnored True and the user has requested that the StandardsCheckerController not display ignored errors, then ShowCheckerErrorWithFixOptions sets HandlingChoice to msdStandardsCheckerReplaceChoiceSkip and returns.

If the standards checker calls this function when interactive mode is not being used, then ShowCheckerErrorWithFixOptions sets SelectedFix to DefaultValue. In this case, ShowCheckerErrorWithFixOptions sets HandlingChoice to msdStandardsCheckerReplaceChoiceFix if the ReplaceOptions indicates that the program can fix problems and the user has indicated that he wants the problems fixed. Otherwise, it sets HandlingChoice to msdStandardsCheckerReplaceChoiceSkip.

Here is sample of the dialog box that the StandardsCheckerController displays. In this example, each field contains a string that is similar to the name of the argument used to specify the contents of the field. The code used to display this dialog box follows the dialog box.

The following methods produce the dialog box that is displayed above: 


Sub ShowSampleMessage()
    Dim choice As MsdStandardsCheckerReplaceChoice ' Output based on button pressed
    Dim selectedFix As Long ' output based on selected row in options dialog box
    Dim columnLabels(0 To 1) As String
    Dim fixes(0 To 2, 0 To 1) As String

    StandardsCheckerController.ShowCheckerStatus "Example Status Message"

    columnLabels(0) = "Example Column 0"
    columnLabels(1) = "Example Column 1"

    fixes(0, 0) = "Example Row 0, column 0":  fixes(0, 1) = "Example Row 0, column 1"
    fixes(1, 0) = "Example Row 1, column 0":  fixes(1, 1) = "Example Row 1, column 1"
    fixes(2, 0) = "Example Row 2, column 0":  fixes(2, 1) = "Example Row 2, column 1"

    ' The first 2 arguments are output.
    StandardsCheckerController.ShowCheckerErrorWithFixOptions choice, selectedFix, _
        "The Problem Description", _
        "Fixes List Box Label", columnLabels, fixes, 0, _
        msdStandardsCheckerReplaceOptionCanFix Or msdStandardsCheckerReplaceOptionCanIgnore, _
        False
End Sub
'
'   ShowCheckerErrorWithFixOptions calls this to get the data to display
'   in the bottom section of the dialog box. It also calls this everytime the
'   user selects a different row in the list of fixes.
'
Private Sub IStandardsChecker_GetFixDetail(Fixes() As String, ByVal selectedFix As Long, FixPropertiesLabel As String, FixProperties() As String)
    FixPropertiesLabel = "VBA Example Fix Properties"
    ReDim FixProperties(0 To 1, 0 To 2)

    FixProperties(0, 0) = "First example property"
    FixProperties(0, 1) = "Current 1"
    FixProperties(0, 2) = "Expected 1"

    FixProperties(1, 0) = "Second example property"
    FixProperties(1, 1) = "Current 2"
    FixProperties(1, 2) = "Expected 2"
End Sub
