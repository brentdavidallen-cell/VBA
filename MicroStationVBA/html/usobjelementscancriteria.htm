ElementScanCriteria Object

                                                     

Use an ElementScanCriteria object as an argument to Scan to limit the set of objects generated by scanning. The scanner can screen out elements much faster than a Visual Basic program can, so it normally is best make the ElementScanCritieria as restrictive as possible.

Remarks

A program that runs outside of MicroStation or needs to create an ElementScanCriteria object in another MicroStation process should use app.CreateObjectInMicroStation ("MicroStationDGN.ElementScanCriteria") where app is an object reference to the Application object for that process. If a program passes an ElementScanCriteria object to Scan, then it is essential that the object comes from the same process where the Scan will run.

Using the expression new ElementScanCriteria in a VBA program running in MicroStation works correctly to create an ElementScanCriteria object in the current process. Therefore, it is the proper thing to do in a VBA application. However, using the expression new ElementScanCriteria to create an ElementScanCriteria object from a separate process while there are multiple copies of MicroStation running may create an object for the wrong process.
