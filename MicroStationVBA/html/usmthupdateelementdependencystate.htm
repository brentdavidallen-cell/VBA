UpdateElementDependencyState Method

                          

Causes MicroStation to update its internal dependency state.

Syntax

object.UpdateElementDependencyState

The UpdateElementDependencyState method syntax has these parts:

Part Description
object A valid object.

Remarks

If a program moves an element that has dependent elements, the dependent elements will not respond until the program calls UpdateElementDependencyState or until another command starts.

It may be necessary to call UpdateElementDependencyState after the initial reference of a model from a design file loaded using OpenDesignFileForProgram. It is only necessary if the program will perform some operation that needs the dependency information. For example, a program that needs to find the tags associated with an element from a design file opened with OpenDesignFileForProgram should call UpdateElementDependencyState.

Programs driving MicroStation by using the CadInputQueue normally do not need to call UpdateElementDependencyState. A program should not call UpdateElementDependencyState unless it implements the current command, i.e., it called StartPrimitveCommand or StartLocateCommand.
