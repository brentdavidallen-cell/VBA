AddCoincident Method

                          

Attaches a reference file coincidentally.

Syntax

Set Attachment = object.AddCoincident (FileSpecification, ModelName, LogicalName, Description [, DisplayImmediately])

The AddCoincident method syntax has these parts:

Part Description
object A valid object.
FileSpecification A String expression. Specifies the name of a reference file. The drive, directory name, and extension are all optional.
ModelName A String expression. The name of the model to attach. If vbNullString is specified, AddCoincident attaches the default model.
LogicalName A String expression. The logical name for the attachment. The logical name must be unique.
DisplayImmediately Optional. A Boolean expression. Specify True to cause Add to immediately display the attachment. Specify False if the program will scale (see ScaleFactor) or rotate (see Rotate) prior to displaying the reference file. The program can subsequently display the attachment using Redraw.
Description A String expression.

Remarks

The format of FileSpecification is configuration_variable:filename where the configuration_variable: part is optional. The configuration_variable part and the colon must appear together. Do not use one without using the other.

If FileSpecification specifies configuration_variable: then MicroStation records configuration_variable: as part of the Attachment. It then evaluates the configuration variable everytime MicroStation loads the Attachment.

If FileSpecification does not provide a full file specification, the master file path and the environment variables MS_RFDIR and MS_DEF are searched sequentially for the file.
