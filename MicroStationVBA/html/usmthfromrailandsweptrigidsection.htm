FromRailAndSweptRigidSection Method

                          

Initializes this object by sweeping, scaling and spinning the given Section along the given Rail path, orienting each cross section with Rail's Frenet frame (cf. EvaluatePointFrame), modified to keep the binormal vector constant at FixedFrameY.

Syntax

object.FromRailAndSweptRigidSection Rail, Section, FixedFrameY [, ScaleX [, ScaleY [, SpinRadians [, Tolerance]]]]

The FromRailAndSweptRigidSection method syntax has these parts:

Part Description
object A valid object.
Rail A BsplineCurve expression. The path along which Section is swept and spun.
Section A BsplineCurve expression. The cross-sectional profile to sweep and scale along Rail. For best results, Section should be planar.
FixedFrameY A Point3d expression. The vector to preserve in the modified Frenet frame. This is the y-axis of the local coordinate system in which each transformed Section is placed along Rail. For example, to preserve the "up" direction of Section, set FixedFrameY to the local y-axis of Section.
ScaleX Optional. A Double expression. The ending scale factor along the modified Frenet frame's x-axis. The starting scale is 1.0 and progresses linearly with each cross section. If omitted, a default scale of 1.0 is used.
ScaleY Optional. A Double expression. The ending scale factor along the modified Frenet frame's constant y-axis. The starting scale is 1.0 and progresses linearly with each cross section. If omitted, a default scale of 1.0 is used.
SpinRadians Optional. A Double expression. The ending spin angle around the modified Frenet frame's z-axis. The starting spin is zero radians and progresses linearly with each cross section. If omitted, a default spin of 0 radians is used.
Tolerance Optional. A Double expression. This is currently not used.

Remarks

For BsplineSurface objects, if the Rail and Section curves' start tangents are not perpendicular or if Rail does not start in Section's initial osculating plane, then Section is rotated internally so that its start binormal is parallel to Rail's start tangent, and it is translated so that its start point lies on Rail's start point if these points were initially close, or if not, so that the projection of Rail's start point onto Section's initial osculating plane lies on Rail's start point (cf. EvaluatePointFrame).

For BsplineSurface objects, the Section is first mapped to the world xy-plane with rotation R(0), given below. For the ith cross section starting with i = 0, this xy-curve is incrementally scaled and spun, and then mapped onto the Rail with rotation R(i) below. Let

y = FixedFrameY,
ti = the tangent vector of Rail at the ith cross section location,
U() = the vector normalization operator, and
a x b = the cross product of vectors a and b.

Then this rotation is given by:

R(i) = Matrix3dFromPoint3dRows(U(y x ti),U(y),U((y x ti) x y))

For BsplineSurface objects, if Rail is G1-continuous (continuity of the normalized tangent) and linear control over scale and spin is not desired, then a method with more options and greater rail approximation accuracy is FromRailsAndSweptSections.
