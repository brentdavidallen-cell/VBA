XDataObject Object

                                                     

Represents a list of XData.

Remarks

With the MicroStationDGN object model it is possible to access XData as an array of XDatum UDT's or as an XDataObject object that holds an array of XData. Prior to MicroStation XM, the MicroStationDGN object model only supported the technique using the array of UDT's. That technique works well with VBA and with VB6, but does not work with .NET because the XDatum UDT may contain a Point3d value in a variant field and .NET's COM interop does not support returning anything that contains a UDT in a variant. Because of this limitation of COM interop it was necessary to add the seemingly redundant technique of providing the XDataObject for accessing the XData.

A programmer working with VBA or VB6 can use either technique. A programmer working with .NET must use the XDataObject if the XData may contain an msdXDataTypePoint, msdXDatumTypeWorldDirection, msdXDatumTypeWorldSpaceDisplacement, or msdXDatumTypeWorldSpacePosition. .NET's COM interop raises an exception as soon as it processes an array with an XDatum UDT. It raises the exception regardless of whether or not the program tries to access the array entry that contains the Point3d.

A .NET program using COM interop can use GetXDatum to retrieve any XDatum that is not a point, but must use GetPoint3d to process an XDatum that contains a Point3d.
