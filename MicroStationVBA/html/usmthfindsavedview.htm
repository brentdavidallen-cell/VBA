FindSavedView Method

                          

Searches this DesignFile for a saved view whose name matches a given pattern. If found, returns the corresponding SavedViewElement object; else returns Nothing.

Syntax

Set SavedViewElement = object.FindSavedView (NamePattern [, PreviouslyFoundSavedView [, Namespace]])

The FindSavedView method syntax has these parts:

Part Description
object A valid object.
NamePattern A String expression. The name of the saved view(s) to find. This name follows the same rules as a pattern given to Visual Basic's Like operator. (Exception: The pattern match is always case-insensitive, regardless of the Option Compare setting.)
PreviouslyFoundSavedView Optional. A SavedViewElement expression. If Nothing (the default), FindSavedView returns the first SavedViewElement whose name matches the given pattern. Otherwise, FindSavedView returns the next SavedViewElement found after this one. Thus, you can call FindSavedView successively to find successive SavedViewElements whose names match the pattern.
Namespace Optional. A String expression. The saved view namespace in which to search. The default value is an empty string, which represents the global namespace (which is the namespace that MicroStation uses in its own Saved Views dialog).

Remarks

See Pattern Matching in Find Methods for a description of the pattern matching.

You can find one specific saved view by simply giving its name, without any wildcard characters.

You can successively find all saved views by giving the name pattern, "*".
